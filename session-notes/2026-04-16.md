## Completed

### Image Optimization Pipeline
- **New script:** `scripts/optimize-images.sh` — idempotent, uses `cwebp` + `convert` (ImageMagick) to regenerate responsive variants from `images/originals/`
- **New folder:** `images/originals/` — preserves source files so variants can be regenerated at any time
- Variants produced (all WebP):
  - `about-hero-{800,1600,2560}.webp` — was 6.3 MB single file → now 28/72/168 KB
  - `services-hero-{800,1600,2560}.webp` — was 19 MB → now 40/144/332 KB
  - `about-steps-{600,1200}.webp` — was 1.4 MB JPG → now 44/136 KB
  - `logo-200.webp` — was 869 KB 1024×1024 → now 4 KB 200×200
  - `logo-text-200.webp` — new transparent-background variant for nav use only (7.5 KB)

### Responsive Hero Backgrounds
- Replaced inline `style="background-image: url(...)"` with CSS utility classes `.hero--bg-about` and `.hero--bg-services` using `image-set()` + a 768px media query
- Added media-scoped `<link rel="preload">` hints so the browser preloads exactly the variant CSS will use (800w under 768px, 1600w above)
- Applied to `index.html`, `about.html`, `services.html` heroes

### Calendar Facade (replaces inline TidyCal on index/about/services)
- Zero-JS, zero-image CSS component — renders a stylized weekly grid with "Schedule a Consultation →" CTA
- Uses container queries (`cqi` units) so typography scales with the column width, not viewport — fits cleanly at any breakpoint
- CTA reuses existing `data-booking` hook → opens the `booking-modal.js` TidyCal modal on click
- Removed the live TidyCal embed iframe + its `embed.js` script from all three pages (~2 MB JS eliminated from initial page load, including reCAPTCHA + Stripe that TidyCal pulled in)

### Map Facade (replaces Google Maps iframes)
- Static screenshot at `images/map-mckinney.webp` (60 KB) with CSS-positioned address pill + "Open in Google Maps →" pill overlay
- Applied to `index.html` (Get In Touch section) and `contact.html` (Visit Our Office section)
- Click → opens native Google Maps in a new tab (better mobile UX than the old inline iframe)
- Saves ~350 KB of Maps JS + ~230 ms main-thread work per page

### Script Loading Cleanup
- Deleted the `G-XXXXXXXXXX` placeholder `<script>` tag from all 9 pages — was downloading 133 KB of useless JS. Real analytics tag is injected at deploy time via Netlify Snippet Injection (Site settings → Build & deploy → Post processing)
- Deleted the matching placeholder `gtag('config', ...)` block in `js/analytics.js`. Custom event tracking (scroll depth, CTA clicks, form submits) still fires via `dataLayer.push` — GTM picks it up when present
- Added `defer` to `<script src="js/analytics.js">` in all 9 pages (was loading synchronously, blocking render)
- Made `animations.css` non-blocking via `<link rel="preload">` + onload-swap pattern
- Added TidyCal preconnect hints so the booking modal opens faster on click

### Nav Logo Fade-on-Scroll
- Nav uses `images/logo-text-200.webp` (transparent background)
- CSS gives the logo a cream (`#fdfbf7`) `background-color` that fades to transparent when the JS-controlled `.is-scrolled` class is applied to the navbar (at the same threshold where the white header bar fades in)
- Footer logo uses the original `logo-200.webp` (cream bg) with softened 4px corners

### Form Accessibility
- Added `autocomplete` attributes to quiz form fields (`sofCity` → `address-level2`, `sofBestTime` → `off`) on `index.html` and `second-opinion.html`
- Added `autocomplete="off"` to all inputs in the hidden Netlify form declarations to stop browsers from offering autofill on invisible fields

### Second Opinion Nav Button
- Changed nav `"Second Opinion Assessment"` button across all 9 pages from `href="second-opinion.html"` to `href="second-opinion.html#sofQuizCard"` — deep-links directly to the quiz card, skipping the page hero

### Calendar Facade CTA Hover
- Global `a:hover { color: var(--navy-700) }` rule was overriding the button's white text — making the label blend into the navy-800 background on hover. Explicitly re-asserted `color: #fff` and `background: var(--navy-800)` in the `.calendar-facade__cta:hover` rule so only the transform/shadow animate

### Cleanup
- Deleted 10 unreferenced backup images from `/images/` (`about-hero-backup.jpg`, `*-original.jpg`, etc.) — ~8 MB of repo weight. Sources preserved in `images/originals/`

## Pending / Future Improvements

Captured in detail in `docs/superpowers/specs/2026-04-16-psi-perf-fix-design.md` §5. Quick list:

1. **Defer GTM until first interaction** — biggest remaining mobile TBT win. Netlify Snippet Injection change, not a repo change
2. **Audit Microsoft Clarity + Facebook Pixel usage** — remove if not actively used
3. **Move tracking behind cookie consent** — privacy win + perf win for decliners
4. **Switch hero sections from `background-image` to `<picture>` / `<img>`** — unlocks `fetchpriority="high"` more effectively
5. **Code-split `second-opinion.js`** — currently loads on every page, only used on index + second-opinion
6. **Subset / self-host Google Fonts** — save ~86 KB and eliminate cross-origin connection
7. **Make render-blocking CSS (styles.css + second-opinion.css) non-blocking** — same preload-swap pattern as animations.css (est. 430 ms desktop / 1,380 ms mobile savings flagged in PSI)
8. **Fix accessibility regressions** — footer contrast, h4 heading order (pre-existing), calendar facade aria-hidden scope
