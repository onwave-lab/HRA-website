## Completed

### Second Opinion Assessment — Full Integration
- **New page:** `second-opinion.html` — standalone quiz page with site navbar, navy page header, embedded quiz, and full site footer

- **New JS:** `js/second-opinion.js` — quiz logic in IIFE with state management, gap scoring, Netlify form submission, bot prevention, and TidyCal booking prefill

### Quiz Embedded in 3 Locations
1. **Homepage** (`index.html`) — full quiz embedded inline in a dedicated section (replaced redirect CTA), with section header "Not Sure If Your Portfolio Has Gaps?"
2. **Standalone page** (`second-opinion.html`) — dedicated quiz page with its own SEO, structured data, and page header
3. **Exit intent popup** (all other pages) — full quiz injected into the exit popup on services, about, contact, client-portal, disclosures, and 404 pages. CSS and JS loaded dynamically. Skipped on homepage (quiz already embedded) and second-opinion page

### Exit Intent Popup Replaced
- Removed the old TidyCal calendar popup
- Replaced with full embedded Second Opinion Assessment quiz
- Popup content area widened (700px) and scrollable (90vh)
- Quiz intro screen serves as the hook — "Before You Go" eyebrow

### Navigation Updates
- **Nav CTA button** added across all 9 pages: secondary outline button ("Second Opinion / Assessment" on two lines) positioned left of "Book a Meeting"
- CSS handles all states: transparent navbar, scrolled navbar, non-transparent navbar, and mobile menu
- Footer "Second Opinion Assessment" link removed from all pages (redundant with header button)

### Form Submission — Dual Netlify Forms
- **`second-opinion`** — receives submissions when consent checkbox is checked (default)
- **`second-opinion-no-contact`** — receives submissions when consent is unchecked
- Both forms include: contact info (first/last name, email, phone, city, best time), all 8 quiz answers, gap count, result level, and consent status
- Hidden Netlify form declarations on `index.html` and `second-opinion.html` for build-time detection
- Submissions sent via background `fetch()` — results display immediately regardless
- **Post-deploy setup required:** Add email notifications in Netlify dashboard for both forms with different subject lines

### Bot Prevention (Quiz)
- **Honeypot fields:** Two hidden inputs (`website_url`, `email_confirm`) added to quiz contact form in all 3 locations
- **Time gate:** 15-second minimum before form submission is accepted (quiz realistically takes 1–3 minutes)
- **Interaction signal:** Requires at least one human interaction event (mouse, scroll, touch, keypress)
- **Graceful degradation:** Bots still see results but form data is silently blocked from Netlify

### TidyCal Booking Prefill
- After quiz completion, `window.sofBookingPrefill` is set with user's full name and email
- Booking modal (`booking-modal.js`) reads this and appends `?name=...&email=...` to the TidyCal iframe URL
- Works in all 3 quiz locations (homepage, standalone page, exit popup)
